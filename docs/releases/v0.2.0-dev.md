# Aby55 v0.2.0-dev

This development release upgrades Aby55 from the first public GitHub build into a more stable local performance firmware for M5Stack Tab5.

## Highlights

- Added a lightweight DSP/control foundation for safer real-time audio: smoothed controls, a gate-driven bass ADSR, DC blockers, drive stages, and soft limiters.
- Improved bass behavior so release can return to idle cleanly, with safer drive/FX ranges and smoother parameter movement.
- Refined drum synthesis with stronger kick punch, layered snare body/noise, and more metallic hi-hat sources while keeping the existing DRUM SYN controls.
- Added real-time audio guardrails: block timing, overrun counters, load-shed state, and a hard cap on rendered drum voices.
- Added ES8388 output observability and profiles for speaker/headphone behavior without reopening the codec on headphone insertion.
- Made MIX page H/M/L EQ boosts more audible and ensured they are not bypassed by audio load-shed.
- Disabled unused runtime hardware paths such as camera power for this firmware's current scope.
- Switched from the ESP-IDF default 1MB app partition to a custom 8MB factory app partition for more room to grow.

## Validation

- `python -m unittest tests.test_project_spec -v`: 26 tests passing.
- `idf.py build`: passing.
- Build output confirms `Smallest app partition is 0x800000 bytes`.
- Hardware boot log confirmed the 8MB factory partition and reached `tab5_drummer: app ready`.

## Notes

This is still a development release. It is intended for the current Aby55 Tab5 hardware baseline and remains experimental firmware, not an official M5Stack or Espressif product.
